# 1小时策略信号分类及使用优化分析

## 1. 文档目的

基于以下源码，对当前 1 小时策略实际生成的信号进行分类梳理，并分析各分类对应的交易方式、跟踪方式、止损逻辑，以及在“策略使用层”是否存在优化空间。

- 主策略文件：`E:\code\vnstudio\vnstudio\Lib\site-packages\vnpy\app\cta_strategy\strategies\dnn_boll_strategy.py`
- 状态常量文件：`E:\code\vnstudio\vnstudio\Lib\site-packages\vnpy\trader\constant.py`
- 数据结构文件：`E:\code\vnstudio\vnstudio\Lib\site-packages\vnpy\trader\object.py`


## 2. 结论摘要

当前 1 小时策略的“信号”并不是单纯的多空提示，而是一个以 `ActorStatus` 为核心的分类体系。这个分类体系已经隐含了以下内容：

1. 信号方向：做多或做空。
2. 信号来源：日线主导、小时长期趋势、小时中期趋势、小时短期突破、平静突破、趋势转折、反手反转。
3. 信号阶段：预备信号（`A_PRE_*`）或正式激活信号（`A_*`）。
4. 交易方式：顺趋势跟踪、短突破试单、转折跟随、持仓反手。
5. 止损方式：不同分类使用不同均线轨道、不同浮亏保护、不同保本移动逻辑。

当前“分类已经存在，但未被直接展示”的根本原因是：

1. `ActorStatus` 被保存到了 `StateInfoData`，但没有进入 `PredictData`。
2. `PredictData` 里保存的是 `evlMomentum`、`furMomentum`、`stopLossPrice`、`calMomentum` 等数值，缺少“分类标签”字段。
3. 因此如果外部只看预测/信号表，而不联动状态表，就只能看到动量和止损价，看不到信号属于哪一类。

结论上，这套 1 小时策略已经具备“分类信号系统”的雏形，后续最值得优先优化的不是重新改造核心逻辑，而是先把现有分类显式化、标签化、分层使用。


## 3. 当前策略的分类主轴

## 3.1 分类核心字段

策略内部真正代表信号类别的是 `ActorStatus`。

其编码规律很清晰：

1. 正数表示偏多，负数表示偏空。
2. `A_PRE_*` 表示预备状态，只是“候选信号”，还不能直接理解为最终可执行信号。
3. 去掉 `PRE` 后的正式状态，才是实际进入可交易/可持仓管理阶段的信号。
4. 数值大小同时隐含了分类层级：
   - `±1111` 一类：日线主导信号
   - `±193` 一类：小时长期均线信号
   - `±151/141/131/121/111`：小时中期趋势分级信号
   - `±105`：小时短期突破信号
   - `±103`：趋势转折跟随信号
   - `±102`：反手强化信号
   - `±101`：平静期突破信号

## 3.2 预备信号与正式信号

这套策略不是“条件满足立即开仓”，而是两段式：

1. 先进入 `A_PRE_*` 预备状态。
2. 再结合 `evlMomentum` 与 `curThreshold` 比较，才升级为正式状态。
3. 正式状态形成后，再结合 DNN 预测过滤、复合动量过滤、止损条件，决定是否真正开仓或持仓。

因此，“信号分类”本质上是一个状态机，不是单次 if 判断。


## 4. 当前 1 小时策略实际会生成的主要分类

下面只列出在 `dnn_boll_strategy.py` 中实际参与生成与交易执行的分类；`constant.py` 中还有一些常量定义，但在该策略里没有真正落地使用，不建议在展示层全部暴露，避免混淆。

## 4.1 日线主导趋势类

- 多头：`A_PRE_UT1_UP_UPLINE_D = 1011`，`A_UT1_UP_UPLINE_D = 1111`
- 空头：`A_PRE_DT1_DN_DNLINE_D = -1011`，`A_DT1_DN_DNLINE_D = -1111`

分类含义：

1. 由日线 20 日均线及其轨道主导。
2. 同时要求小时中期或长期位置也同向配合。
3. 属于策略中级别最高的一类趋势信号。

交易方式：

1. 偏“主趋势跟踪”。
2. 日线方向先成立，小时线配合确认后再允许入场。
3. 当小时级已有趋势信号，并且日线继续强化时，可升级为日线主导状态。

止损方式：

1. 默认使用日线中轨到上/下轨之间的四分之一位置作为防守线。
2. 如果偏离小时中期均线过大，会把 `defender` 切到 1，改为使用小时中期均线防守。
3. 若开仓后反向运动过大且浮亏超过 1%，会触发更紧的防守。
4. 浮盈超过 2% 后，会把成本价抬到保本附近。

使用特征：

1. 级别高。
2. 持仓逻辑偏稳。
3. 更适合作为“主信号”。


## 4.2 小时长期均线趋势类

- 多头：`A_PRE_UT1_UP_UPLINE_L = 93`，`A_UT1_UP_UPLINE_L = 193`
- 空头：`A_PRE_DT1_DN_DNLINE_L = -93`，`A_DT1_DN_DNLINE_L = -193`

分类含义：

1. 由 100 小时均线相关轨道主导。
2. 要求 50 小时均线与 100 小时均线形成明确顺序关系。
3. 相比普通小时趋势，这类更强调大级别小时结构。

交易方式：

1. 偏“中长期跟随”。
2. 比日线类弱一级，但比普通小时趋势更稳定。
3. `defender` 会决定防守是放在 100 小时轨道，还是切到 50 小时轨道。

止损方式：

1. 默认使用长期中轨到上/下轨之间的五分之一位置。
2. 若 `defender = 1` 且持仓已转弱，还会切换为小时中期轨道防守。

使用特征：

1. 适合过滤掉太短的波动噪音。
2. 更适合作为“次主信号”。


## 4.3 小时中期趋势分级类

对应分类：

- 1%-2% 趋势：`±111`
- 2%-3% 趋势：`±121`
- 3%-4% 趋势：`±131`
- 4%-5% 趋势：`±141`
- 5% 趋势：`±151`

其中预备状态分别是：

- `±11`、`±21`、`±31`、`±41`、`±51`

分类含义：

1. 由 50 小时均线从近期低点/高点偏移的幅度划分。
2. 并不是“数值越大越好”，而是代表趋势已经走到不同成熟阶段。
3. 本质上是顺着小时中期趋势做跟随。

交易方式：

1. 都属于顺趋势跟踪。
2. 趋势越成熟，`majorMomentum`、`supMomentum` 的权重组合不同。
3. 早期趋势更强调启动确认，晚期趋势更强调跟踪和保护利润。

止损方式：

1. 普通中期趋势类，使用小时中期轨道：
   - 多头：`midLine + (upLine - midLine) / 5`
   - 空头：`midLine + (dnLine - midLine) / 5`
2. 5% 趋势类是特例，止损更紧：
   - 多头：`/3`
   - 空头：`/3`

重要理解：

1. `151/-151` 不是“最强信号”，而是“趋势已经较成熟的信号”。
2. 这类信号更像“追踪型信号”，而不是最早启动型信号。
3. 所以它的止损更紧，体现的是利润保护，不是更激进加仓。


## 4.4 小时短期突破类

- 多头：`A_PRE_UT0_UP_UPLINE_S = 5`，`A_UT0_UP_UPLINE_S = 105`
- 空头：`A_PRE_DT0_DN_DNLINE_S = -5`，`A_DT0_DN_DNLINE_S = -105`

分类含义：

1. 由 30 小时均线相关的短期结构触发。
2. 涨跌幅要求较小，属于短波段启动或局部突破。
3. 要求前期走势较平滑，且日线不能处于强烈反向压制中。

交易方式：

1. 更偏“试探性趋势跟随”。
2. 相比日线类和长期类，级别更低，噪音更大。
3. 对 DNN 过滤更敏感。

止损方式：

1. 若开仓后即浮亏，直接用固定比例保护：
   - 多头：`ftBuyPrice * 0.992`
   - 空头：`ftSellPrice * 1.008`
2. 否则使用短期轨道中轨到上下轨的五分之一位置。

使用特征：

1. 更适合当“辅助信号”。
2. 不适合脱离大级别环境单独重仓使用。


## 4.5 平静期突破类

- 多头：`A_PRE_QUIET_UP_BREAK = 1`，`A_QUIET_UP_BREAK = 101`
- 空头：`A_PRE_QUIET_DOWN_BREAK = -1`，`A_QUIET_DOWN_BREAK = -101`

分类含义：

1. MA50 较平，STD10/STD60 较低，说明市场处于相对安静状态。
2. 在低波动环境中向上轨/下轨突破。
3. 属于“低波动突破”。

交易方式：

1. 偏突破跟随。
2. 信号弹性大，但也更容易出现假突破。
3. 代码里对这类信号会放宽 `TRACK_STEP`、`EVALUATE_STEP`，给它更多确认时间。

止损方式：

1. 归入普通中期趋势类的防守逻辑。
2. 但由于其本质是低波动突破，实际使用时应把它视为更容易失真的分类。

使用特征：

1. 假突破风险较高。
2. 建议只在大级别方向一致时提高权重。


## 4.6 趋势转折类

- 多头：`A_PRE_UT0_UP_UPLINE_T = 3`，`A_UT0_UP_UPLINE_T = 103`
- 空头：`A_PRE_DT0_DN_DNLINE_T = -3`，`A_DT0_DN_DNLINE_T = -103`

分类含义：

1. 用于识别由下跌向上涨、或由上涨向下跌的转折。
2. 依赖 `runningStage` 判断当前处于上行初期还是下行初期。
3. 会根据 `slopeH/slopeL` 给出 `prepHoldPeriod`，即准备持有期。

交易方式：

1. 偏“转折后跟随”。
2. 比普通趋势信号更早，但也更容易失败。
3. 不只是看突破，还看转折阶段与动量关系。

止损方式：

1. 基本仍使用轨道防守。
2. 但如果超过 `prepHoldPeriod` 后仍只有小利润，就会把止损收得非常紧，接近当前价。

使用特征：

1. 属于激进型跟随信号。
2. 更适合做“早期试单”，不适合视作最终强确认。


## 4.7 反手强化类

- 多头：`A_UT0_UP_UPLINE_T0 = 102`
- 空头：`A_DT0_DN_DNLINE_T0 = -102`

分类含义：

1. 这是趋势转折类中的强化变体。
2. 不是普通候选状态，而是“原方向已有持仓/状态时，出现足够强的反向转折”。
3. 代码后面会把它直接转换成 `A_UT0_UP_UPLINE_T` 或 `A_DT0_DN_DNLINE_T` 并执行反手开仓。

交易方式：

1. 偏“直接反手”。
2. 是当前策略中最激进的一类。
3. 不只是提示，而是有真实的反手开仓语义。

止损方式：

1. 后续会转入 `T` 类信号管理。
2. 但由于它是反手开仓，本质上更适合快速验证，不适合长时间容忍回撤。

使用特征：

1. 灵敏度高。
2. 波动敏感。
3. 如果后续做展示，应单独标注为“反手信号”，不能与普通顺趋势信号混在一起。


## 5. 分类与交易方式的对应关系

可以把当前 1 小时策略的正式信号分成 4 个使用层级。

## 5.1 A 类：主趋势信号

包括：

1. `±1111` 日线主导
2. `±193` 小时长期均线主导

特点：

1. 趋势级别高。
2. 防守线相对稳定。
3. 更适合作为主仓或主参考信号。

## 5.2 B 类：标准顺趋势信号

包括：

1. `±111`
2. `±121`
3. `±131`
4. `±141`
5. `±151`

特点：

1. 都属于小时中期趋势跟随。
2. 是当前策略最常规的趋势交易层。
3. 其中 `±151` 更偏成熟趋势追踪，止损更紧。

## 5.3 C 类：突破试单信号

包括：

1. `±105`
2. `±101`

特点：

1. 启动更早。
2. 噪音更大。
3. 更适合做提示或轻仓试单，而不是直接当成最高优先级信号。

## 5.4 D 类：转折/反手信号

包括：

1. `±103`
2. `±102`

特点：

1. 偏转折交易。
2. 适合捕捉早期切换，但失败概率也更高。
3. `±102` 应视为最高敏感度、最高攻击性的信号。


## 6. 为什么当前系统里“分类不展示”

从数据结构看，当前系统已经保存了分类所需信息，但分散在不同对象里。

## 6.1 已保存的分类信息

`StateInfoData` 中已经保存：

1. `ActorStatus`
2. `defender`
3. `prepHoldPeriod`
4. `runningStage`
5. `dnnTrigger`
6. `openMomentum`
7. `extremeMomentum`
8. `extremeCalMomentum`

这说明策略运行态的分类、风格、跟踪方式，实际上已经落库。

## 6.2 已保存但不含分类标签的信息

`PredictData` 中保存：

1. `evlMomentum`
2. `furMomentum`
3. `furMomentum0/1/2`
4. `curThreshold`
5. `stopLossPrice`
6. `calMomentum`
7. `calThreshold`

这里能看出强弱、止损和 DNN 预测，但看不出“这是哪一类信号”。

## 6.3 根本原因

当前展示缺口不在策略生成层，而在“读数与展示层”：

1. 生成层有分类。
2. 状态层保存了分类。
3. 预测层没有带分类标签。
4. 如果外部只读预测表，就无法直观看到分类。


## 7. 使用层面的优化空间

这里讨论的是“策略使用优化”，不是改动核心信号逻辑。

## 7.1 优化点一：先把分类显式展示出来

这是最优先的优化点。

建议后续在信号查看层显式展示以下字段：

1. `ActorStatus`
2. `分类标签`
3. `方向`
4. `级别`
5. `来源周期`
6. `是否 PRE`
7. `止损价`
8. `defender`
9. `prepHoldPeriod`
10. `dnnTrigger`

这样外部看到的就不再只是“多/空”，而是：

1. 日线主导多头
2. 小时长期空头
3. 中期 5% 趋势多头
4. 短期突破空头
5. 转折反手多头

这会大幅提高信号可解释性。


## 7.2 优化点二：不要把所有正式信号等权使用

当前系统最适合做的不是“有信号就一样对待”，而是分层使用。

建议的使用优先级：

1. 第一优先级：`±1111`、`±193`
2. 第二优先级：`±131`、`±141`、`±151`
3. 第三优先级：`±111`、`±121`
4. 第四优先级：`±105`、`±101`
5. 第五优先级：`±103`、`±102`

原因：

1. 主趋势类更稳定。
2. 中期趋势类是标准跟踪层。
3. 短突破和平静突破更容易受噪音影响。
4. 转折和反手类最敏感，最适合单独管理。


## 7.3 优化点三：不要把“幅度更大”误读为“质量更高”

这是当前策略最容易被误用的地方。

例如：

1. `151` 并不代表一定优于 `111`。
2. `151` 代表趋势已发展到较成熟阶段，因此止损更紧，更偏利润保护。
3. `111` 反而更接近趋势早期确认。

所以后续如果做展示或实盘使用，最好把“趋势级别”与“成熟度”区分开，不要只按数字大小排序。


## 7.4 优化点四：单独标识“反手类”

`102/-102` 与普通顺趋势信号完全不是同一种交易动作。

它们的特点是：

1. 发生在原方向状态已存在的背景下。
2. 语义上是“直接反手”。
3. 风险特征与普通趋势跟随不同。

因此后续无论做展示还是做人工使用，都应该把它们单独列出来，而不是跟普通多空提示混排。


## 7.5 优化点五：DNN 过滤结果应和分类一起看

当前策略里，DNN 不是主分类器，而是第二层过滤器。

所以更合理的使用方式不是只看 `furMomentum`，而是看：

1. 基础分类是什么
2. 当前 `evlMomentum` 是否过阈值
3. `furMomentum` 是否同向
4. `calMomentum` 是否同向
5. `dnnTrigger` 是否已经触发过退出或禁止开仓

其中：

1. A/B 类信号更适合把 DNN 作为确认器。
2. C/D 类信号更适合把 DNN 作为强过滤器。


## 8. 对后续 15/30 分钟扩展的启发

如果后续把这套思路扩展到 15 分钟和 30 分钟，建议保留的不是“完全复制所有阈值”，而是保留这套分类框架：

1. 主趋势类
2. 标准趋势类
3. 短突破类
4. 平静突破类
5. 转折类
6. 反手类

然后把：

1. 趋势参考均线
2. 轨道宽度
3. 动量阈值
4. 准备持有期
5. 止损公式

改成适配 15 分钟、30 分钟节奏的版本。

也就是说，当前 1 小时策略已经提供了一套“分类框架模板”，后续 15/30 分钟更适合做的是参数和周期映射，而不是从零重新定义信号类别。


## 9. 最终判断

当前 1 小时策略并不是“只有一个信号逻辑”，而是已经形成了一套完整的分层信号体系：

1. 日线主导趋势
2. 小时长期趋势
3. 小时中期趋势分级
4. 小时短期突破
5. 平静突破
6. 趋势转折
7. 反手强化

从策略使用层面看，最大的优化空间有三点：

1. 把 `ActorStatus` 分类显式展示出来。
2. 按信号类别分层使用，而不是所有信号一视同仁。
3. 将“趋势成熟度”“反手属性”“DNN 过滤结果”一起呈现，而不是只展示一个买卖方向。

如果只做一件最有价值的事情，建议优先做：

1. 用 `StateInfoData.ActorStatus` 建立一份可读的“信号分类字典”；
2. 再把它与 `PredictData.stopLossPrice / evlMomentum / calMomentum` 联合展示。

这样即使不改动核心策略逻辑，也能显著提升当前 1 小时策略信号的可解释性与可用性。

# 1小时策略固定参数归类与影响分析

## 1. 文档目的

针对 1 小时策略文件 `E:\code\vnstudio\vnstudio\Lib\site-packages\vnpy\app\cta_strategy\strategies\dnn_boll_strategy.py`，梳理其中已经存在的固定参数、自定义参数、硬编码阈值及其影响，便于后续做参数优化、实盘修正和 15/30 分钟策略扩展时参考。

本文件重点回答 4 个问题：

1. 当前策略里有哪些参数是真正可直接配置的。
2. 哪些参数虽然影响很大，但目前是写死在代码里的。
3. 每类参数影响的是“信号生成”“信号过滤”“止损退出”还是“DNN 二次筛选”。
4. 哪些参数最值得优先优化。


## 2. 先给结论

当前 1 小时策略的参数可以分为两大层：

1. 显式参数：已经在 `parameters` 中暴露，可直接作为策略参数使用。
2. 隐式参数：大量直接写死在代码中的比例、阈值、窗口长度、权重和止损分数。

从后续优化价值来看，最值得优先关注的并不是成交相关参数，而是以下 5 组：

1. 波动过滤阈值组：决定信号多不多、噪音大不大。
2. 趋势分级阈值组：决定策略把行情划到哪一类。
3. 止损保护阈值组：决定回撤容忍度和利润保护速度。
4. DNN 反向过滤阈值组：决定好信号会不会被过滤掉，坏信号会不会漏进来。
5. 复合动量 `calMomentum` 权重组：决定 DNN 和传统动量各占多大话语权。

如果后续只做一轮优化，建议优先做：

1. `curThreshold` 相关阈值
2. 趋势分级百分比阈值
3. 强制止损与保本阈值
4. DNN 反向过滤阈值
5. `calco`


## 3. 参数总分类

## 3.1 第一类：策略显式参数

这部分参数已经在 `parameters` 中暴露：

1. `priceCo = 1.0`
2. `calco = 0.8`
3. `bandOffset = 1.25`
4. `sendVolume = 1`
5. `priceOffset = 3`
6. `retryTimes = 3`

这类参数的特点是：

1. 可以直接作为策略输入进行修改。
2. 影响范围清晰。
3. 适合优先做外部配置化管理。


## 3.2 第二类：行情结构固定参数

这类参数定义了均线、标准差、ATR 的观察周期与轨道结构，例如：

1. 小时线 SMA：5、10、20、30、40、50、100
2. 小时线 ATR：10、30、60
3. 小时线 STD：10、30、50、60、100
4. 日线 SMA：5、10、20
5. 日线 STD：10、20

这类参数的特点是：

1. 决定策略观察的市场结构。
2. 影响趋势识别的基础节奏。
3. 不建议在没有充分回测前频繁调整。


## 3.3 第三类：趋势分类阈值参数

这类参数用于决定行情属于哪一种趋势或突破分类，例如：

1. 1%、2%、3%、4%、5% 的小时趋势分级阈值
2. 1.5% 的小时长期趋势阈值
3. 1% 的日线趋势阈值
4. 0.5% 的小时短突破阈值

这类参数的特点是：

1. 决定分类标签。
2. 决定信号的密度与成熟度。
3. 是后续策略分类优化的核心。


## 3.4 第四类：波动过滤参数

这类参数用于过滤杂波、平静期、极端拉升、过热行情，例如：

1. `0.006`
2. `0.012`
3. `0.0125`
4. `0.015`
5. `0.016`
6. `0.018`
7. `0.022`
8. `0.025`
9. `1.6`

这类参数的特点是：

1. 决定开仓门槛。
2. 决定策略在高波动和低波动下的灵敏度。
3. 是最影响“信号多寡”和“虚假信号数量”的一组参数。


## 3.5 第五类：动量评估参数

这类参数用于计算 `evlMomentum`、`majorMomentum`、`supMomentum`、`calMomentum`。

例如：

1. `0.3 / 0.2 / 0.1` 的传统动量权重
2. `0.12 / 0.14 / 0.16 / 0.18 / 0.2` 的支持动量权重
3. `1.012 / 1.025` 的动量时效放大因子
4. `8 / 16 / 32` 的动量有效阶段长度
5. `calco = 0.8` 的复合动量开仓阈值比例

这类参数的特点是：

1. 决定趋势确认快慢。
2. 决定早期信号和成熟信号的相对权重。
3. 直接影响开仓、续持和提前退出。


## 3.6 第六类：止损止盈参数

这类参数决定防守方式，例如：

1. 轨道分数 `/4`、`/5`、`/3`
2. 固定浮亏保护 `0.992 / 1.008`
3. 反向浮亏 1% 触发紧急止损
4. 浮盈 2% 后移动到保本
5. 偏离中期均线 5% 后切换防守轨道
6. 强制止损 2.5%

这类参数的特点是：

1. 决定回撤容忍度。
2. 决定趋势单能否拿住。
3. 对实盘体验影响极大。


## 3.7 第七类：DNN 过滤与复合动量参数

这类参数用于第二层过滤和二次开仓判断，例如：

1. `revCo` 的 0.5 / 0.55 / slope
2. `0.8 / 0.66 / 0.22 / 0.3 / 0.2`
3. `curThreshold * 2`
4. `curThreshold * 1.6`
5. `20` 根 bar 的有效观察期
6. `10` 根 bar 的重入期
7. `30` 根 bar 的提示有效期

这类参数的特点是：

1. 决定 DNN 会不会太严格。
2. 决定被 DNN 拒绝后的重入难度。
3. 是最容易造成“明明趋势对了但进不去”的部分。


## 3.8 第八类：成交与运行参数

这类参数包括：

1. `sendVolume = 1`
2. `priceOffset = 3`
3. `retryTimes = 3`
4. `initDays = 40 / 100`
5. `training_data_num = 128 / 64`
6. `leading_data_num = 10`

这类参数的特点是：

1. 更偏执行与准备阶段。
2. 对信号质量影响小于前几组。
3. 对实盘委托成功率和初始化完整性有影响。


## 4. 显式参数详细分析

## 4.1 `priceCo = 1.0`

作用：

1. 是整个策略的价格尺度系数。
2. 大量百分比阈值都会乘上它。
3. 包括趋势分类、波动过滤、止损保护、强制止损、保本保护等。

影响：

1. 调大后，所有阈值整体变宽，信号会减少，止损会变松。
2. 调小后，信号会增多，止损会更灵敏，也更容易被噪音触发。

优化建议：

1. 这是全局总开关，不建议轻易调。
2. 除非品种价格特征与设计假设严重不一致，否则优先调局部阈值，不先动它。


## 4.2 `calco = 0.8`

作用：

1. 是复合动量 `calMomentum` 的开仓阈值比例。
2. 当 `calMomentum > curThreshold * calco` 时，才更容易允许多头开仓。
3. 空头同理。

影响：

1. 调大后，DNN+传统动量联合信号会更严格，开仓减少。
2. 调小后，更容易开仓，但 DNN 过滤效果会变弱。

优化建议：

1. 这是 DNN 策略落地时最值得优先调的显式参数之一。
2. 如果出现“信号方向经常对，但进场太少”，优先检查它。


## 4.3 `bandOffset = 1.25`

作用：

1. 作为小时中期 50 小时轨道宽度系数。
2. 影响上轨、下轨位置，从而影响 `curLoc` 的切换。
3. 进一步影响分类生成和止损轨道。

影响：

1. 调大后，轨道更宽，突破更少，信号减少。
2. 调小后，轨道更窄，突破更频繁，信号更多但噪音更大。

注意：

1. 小时长期轨道写死为 `1`
2. 日线轨道写死为 `1.25`
3. 小时短期轨道写死为 `1.5`

所以当前只有中期主轨道是显式可调的，其余周期轨道仍是硬编码。


## 4.4 `sendVolume = 1`

作用：

1. 委托单发送手数。

影响：

1. 不影响信号生成。
2. 只影响每次实际下单量和虚拟仓位变化。

优化建议：

1. 与策略逻辑优化关系较小。
2. 更偏实盘执行配置。


## 4.5 `priceOffset = 3`

作用：

1. 重发委托时，按最小跳动价的 3 个价位进行价格修正。

影响：

1. 会影响挂单成交概率。
2. 不影响信号生成。

优化建议：

1. 适合按不同品种流动性做实盘微调。
2. 不属于本次“信号质量优化”的第一优先级。


## 4.6 `retryTimes = 3`

作用：

1. 委托未完全成交时的最大重试次数。

影响：

1. 影响执行完整性。
2. 不影响信号分类和买卖点逻辑。

优化建议：

1. 与策略信号质量关系低。


## 5. 代码写死但影响较大的参数

## 5.1 轨道宽度参数

当前实际使用：

1. `band_offset = self.bandOffset = 1.25`
2. `band_offset_L = 1`
3. `band_offset_D = 1.25`
4. `band_offset_S = 1.5`

影响：

1. 长期轨道最窄，更容易识别大趋势位置切换。
2. 短期轨道最宽，降低短周期虚假突破。
3. 日线与中期轨道一致，说明设计上把日线与中期都作为主判断层。

优化判断：

1. 如果后续发现短期信号过少，可先看 `band_offset_S = 1.5` 是否过宽。
2. 如果中期信号过多，可先看 `bandOffset = 1.25` 是否偏窄。


## 5.2 趋势分类阈值参数

当前主要阈值：

1. 小时长期趋势：`1.5%`
2. 小时中期趋势分级：`1% / 2% / 3% / 4% / 5%`
3. 日线趋势：`1%`
4. 小时短突破：`0.5%`
5. 趋势阶段切换：`0.5%`

影响：

1. 阈值越低，策略更容易把行情归类为趋势。
2. 阈值越高，策略更偏向只识别已经展开的行情。
3. 这组阈值直接决定 `ActorStatus` 的分类标签分布。

优化判断：

1. 如果发现大多数信号集中在 `UT1/DT1`，而高等级趋势很少，说明分级阈值偏高或品种波动不足。
2. 如果发现高等级趋势太多，说明分级阈值偏低，分类失真。


## 5.3 日线过滤参数

关键阈值包括：

1. `curStd10_D < close * 0.01`
2. `curStd20_D > close * 0.022`
3. 日线平静区定义：`0.006`
4. 20 日均线 20 天涨跌大于 `10%` 且当前偏离大于 `10%` 时，不开仓
5. 10 天参考涨跌大于 `8%` 且当前偏离大于 `10%` 时，不开仓

影响：

1. 这组参数决定日线是否允许小时信号通过。
2. 会显著影响开仓数量。
3. 对趋势过热行情有明显抑制。

优化判断：

1. 如果策略经常在趋势末端不开仓，这是合理现象，说明过热过滤在起作用。
2. 如果策略过于保守，可以优先检查 `10%/8%/10%` 这组限制是否过严。


## 5.4 波动率过滤参数

关键阈值包括：

1. `curAtr60 / curClose < 0.007`
2. `std60Co` 默认 `0.02`
3. `std60Co` 下限 `0.015`
4. `curAtr60 <= close * 0.012`
5. `curStd60 < close * 0.016 / 0.018 / 0.022 / 0.025`
6. `curStd30 < close * 0.016 / 0.018 / 0.02`
7. 低波动补偿系数上限 `1.6`

影响：

1. 决定当前环境是否“适合交易”。
2. 决定 `cond_STD` 是否成立。
3. 决定 `curThreshold` 是否需要被抬高，避免低波动误触发。

优化判断：

1. 这是最影响信号数量的一组参数。
2. 如果出现“行情明显在走，但系统迟迟不给信号”，往往要回头检查这组。
3. 如果出现“横盘里频繁误报”，也要优先检查这组。


## 5.5 TRACK / EVALUATE 参数

当前逻辑：

1. 正常情况下：
   - `TRACK_STEP = 5`
   - `EVALUATE_STEP = 5`
2. 平静突破或低波动时：
   - `TRACK_STEP = 10`
   - `EVALUATE_STEP = 10`

影响：

1. 决定预备状态能在多长时间内升级为正式状态。
2. 时间越短，确认更快，但更容易误判。
3. 时间越长，确认更稳，但更容易错过初段行情。

优化判断：

1. 如果信号太慢，优先检查这组。
2. 如果突破类假信号太多，也应检查这组。


## 5.6 转折类参数

关键阈值：

1. `slopeH < 0.2 / 0.4`
2. `slopeL < 0.2 / 0.4`
3. `prepHoldPeriod = 10 / 8 / 5`
4. `(curClose-curSma10)*0.3 > obvMomentum*0.45`
5. `(curClose-curSma10)*0.3 < obvMomentum*0.36`

影响：

1. 决定转折类信号是否容易触发。
2. 决定转折信号拿多长时间看成败。
3. 对 `T` 类与 `T0` 类信号非常关键。

优化判断：

1. 如果反手类过于频繁，优先检查 `0.45 / 0.36` 与 `prepHoldPeriod`。
2. 如果转折信号经常拿不住，可检查 `10/8/5` 是否过短。


## 6. 动量计算参数归类

## 6.1 观察动量权重

当前基础观察动量：

1. 日线观察动量：`0.3 + 0.3`
2. 小时观察动量：`0.3 + 0.2 + 0.1`

影响：

1. 短均线越重，动量越敏感。
2. 长均线越重，动量越稳。


## 6.2 趋势分类对应的评估动量权重

不同趋势类别有不同组合：

1. 5% 趋势更强调 `SMA60`
2. 1% 趋势更强调 `SMA30`
3. 短趋势更强调 `SMA10`
4. 日线趋势则把小时 `SMA10` 与日线 `SMA20` 混合

影响：

1. 决定各类信号的“速度感”。
2. 越偏长周期，越稳但越慢。
3. 越偏短周期，越快但更容易回撤。


## 6.3 支持动量参数

关键权重：

1. `0.12`
2. `0.14`
3. `0.16`
4. `0.18`
5. `0.2`
6. 转折类 `0.33`
7. 日线辅助 `0.1`

影响：

1. 支持动量越大，越容易在短时逆向波动里保留仓位。
2. 支持动量过大，会造成“明明该止损却还在扛”。

优化判断：

1. 如果策略过早出场，可提高支持动量。
2. 如果策略回撤过大，可降低支持动量。


## 6.4 动量时效参数

关键参数：

1. `1.012`
2. `1.025`
3. `8 / 16 / 32`

影响：

1. 用于放大信号初期，随后再衰减。
2. 这决定策略偏“追初动”还是“追确认”。

优化判断：

1. 1 小时策略如果感觉过慢，可先看这组。
2. 如果前几根 bar 假突破太多，则可能放大过强。


## 7. 止损止盈参数归类

## 7.1 轨道型止损参数

核心分数：

1. 日线主导：`/4`
2. 长期趋势：`/5`
3. 普通趋势：`/5`
4. 5% 成熟趋势：`/3`

影响：

1. `/4` 比 `/5` 更紧。
2. `/3` 最紧，更偏保护利润。

判断：

1. 当前策略对成熟趋势的止损是更紧的，不是更松的。
2. 这意味着它默认认为成熟趋势更应防回吐。


## 7.2 固定比例止损参数

关键参数：

1. 短期多头浮亏保护：`0.992`
2. 短期空头浮亏保护：`1.008`
3. 日线反向浮亏阈值：`1%`
4. 强制止损：`2.5%`

影响：

1. `0.8%` 的短期浮亏保护很敏感。
2. `2.5%` 的强制止损相对更宽。
3. 两者组合形成“短信号快止损、大错单强制止损”的结构。

优化判断：

1. 如果短期信号总被提前洗掉，可先检查 `0.992/1.008`。
2. 如果实盘回撤过大，可先检查 `2.5%` 是否过松。


## 7.3 保本与利润保护参数

关键参数：

1. 浮盈超过 `2%` 后，把保本线提到成本价附近。
2. `txHoldBarCount > 10` 后才正式启用保本保护。

影响：

1. 对趋势延续有利，但对快速回撤也能起到保护。
2. 如果品种波动快，`10` 根 bar 可能偏慢。

优化判断：

1. 如果盈利回吐过大，可优先检查 `2%` 和 `10 bar`。


## 7.4 防守轨道切换参数

关键参数：

1. 日线建仓偏离小时中期均线超过 `5%`，切换防守轨道。

影响：

1. 大幅偏离后，防守从更远的日线轨道切到更近的小时中期轨道。
2. 本质上是在利润较大时加强保护。

优化判断：

1. 如果日线大趋势单总被提前洗掉，可能 `5%` 偏低。
2. 如果利润回撤过大，可能 `5%` 偏高。


## 8. DNN 过滤参数归类

## 8.1 DNN 反向过滤阈值

关键参数：

1. `revCo = 0.5`
2. 当斜率较强时，提升到 `0.55` 或直接取 `slope`
3. `farfurMomentum_avg` 使用最近 `3` 个值均值
4. 预测末值与均值比较使用 `0.8`
5. 无仓位时提前拦截阈值为 `0.66`

影响：

1. 趋势越强，允许 DNN 有更大反向波动。
2. 趋势越弱，DNN 稍有反向就会拦截。

优化判断：

1. 如果 DNN 过于保守，优先看 `revCo` 与 `0.66`。
2. 如果 DNN 过滤不够，优先看 `0.8` 与 `3` 根平均。


## 8.2 DNN 重入参数

关键参数：

1. 提示有效期 `30` bar
2. 重入观察期 `10` bar
3. 价格回撤容忍 `0.04 * close`
4. 重入下限 `0.22 * curThreshold`
5. 回撤容忍带 `±0.3 * curThreshold`
6. 强信号残留条件 `0.6 * curThreshold`

影响：

1. 决定被 DNN 拒绝后，是否还有机会重新进场。
2. 这组参数太严，会导致“出去了就回不来”。
3. 太松，则会导致“刚被过滤又冲进去”。


## 8.3 复合动量权重参数

典型权重：

1. 同向温和：`0.7 + 0.3`
2. 反向博弈：`0.8 + 0.2`
3. 新高/新低附近：`0.9 + 0.1`
4. 无明显优势时：`0.5 + 0.5`
5. 截断阈值：`2 * curThreshold`
6. 截断阈值：`1.6 * curThreshold`

影响：

1. 趋势一致时，DNN 可增强传统动量。
2. 趋势冲突时，DNN 只占 20% 权重，避免过度反向。
3. 这是传统技术指标和模型结果的融合核心。

优化判断：

1. 如果 DNN 价值不明显，可提高同向情况下的模型权重。
2. 如果模型经常扰乱正常趋势，可降低反向情况下的模型权重。


## 9. 训练与标签参数

虽然当前使用的是外部已训练模型，但代码中仍有一批影响样本准备的固定参数：

1. `training_data_num = 128`
2. 高波动或强趋势时缩短到 `64`
3. `leading_data_num = 10`
4. `delayCo_1 = 0.995 / 0.99`
5. `delayCo_2 = 0.99 / 0.96`
6. `furCo = 0.3 / 0.5`
7. 标签衰减 `0.95`
8. 远期动量折算 `0.66`

影响：

1. 这组参数影响的是训练数据标签，不直接影响当前已加载模型的在线推理逻辑。
2. 但如果后续要训练 15 分钟或 30 分钟模型，这组参数就非常关键。

优化判断：

1. 对当前 1 小时“仅使用已有模型”的运行影响较低。
2. 对后续模型重训影响很高。


## 10. 参数优化优先级建议

## 10.1 第一优先级

建议最先优化：

1. 波动过滤阈值：`0.006 / 0.012 / 0.015 / 0.016 / 0.018 / 0.022 / 0.025 / 1.6`
2. 趋势分级阈值：`0.5% / 1% / 1.5% / 2% / 3% / 4% / 5%`
3. 止损参数：`0.992 / 1.008 / 1% / 2% / 2.5% / 5%`
4. DNN 过滤参数：`revCo`、`0.66`、`0.22`、`0.3`
5. `calco = 0.8`

原因：

1. 它们直接影响信号数量、信号质量和实际盈亏体验。


## 10.2 第二优先级

建议第二轮优化：

1. `TRACK_STEP / EVALUATE_STEP`
2. `prepHoldPeriod`
3. 动量权重 `0.3 / 0.2 / 0.1`
4. 支持动量权重 `0.12 ~ 0.2`

原因：

1. 这组更偏风格层调优。


## 10.3 第三优先级

建议最后再调：

1. `sendVolume`
2. `priceOffset`
3. `retryTimes`
4. `initDays`

原因：

1. 它们更多影响执行和初始化，不是核心信号质量问题。


## 11. 最终结论

当前 1 小时策略虽然表面上只有少量参数暴露，但实际上内部存在大量硬编码参数，并且这些参数已经形成了完整的多层控制结构：

1. 第一层：趋势识别
2. 第二层：波动过滤
3. 第三层：动量确认
4. 第四层：DNN 二次过滤
5. 第五层：止损与保本保护

如果后续要做“参数优化”，最重要的不是一次性全改，而是按下面顺序分层推进：

1. 先调波动过滤和趋势分类，让信号分类合理。
2. 再调止损和保本，让持仓体验合理。
3. 再调 DNN 过滤与复合动量，让模型发挥作用但不过度干预。

从实务角度看，当前最值得重点记录、后续最有优化价值的参数组是：

1. `curThreshold` 的形成与补偿参数
2. 各类趋势分级百分比
3. `0.992 / 1.008 / 2.5% / 2% / 5%` 这组风控参数
4. `revCo / 0.66 / 0.22 / 0.3 / calco`

这几组基本决定了：

1. 信号出现得快不快
2. 信号会不会太多
3. 仓位能不能拿住
4. DNN 会不会过度过滤
